## Proposal

Add [`docs/forward_workflow.md`](../forward_workflow.md) as a first‑class governance artifact describing the operational pipeline for the Constructed‑Becoming ecosystem. The document synthesises the lineage chronicle, GitHub Actions agent runs, Copilot agent session history, the Agent Factory templating system, and the current repository identity into a single, coherent, seven‑phase workflow.

Cross‑link the document from `README.md` so it is discoverable alongside other governance and operational documents.

---

## Rationale

The repository's governance corpus described *what* the ecosystem is and *how* agents should behave, but lacked an explicit description of *how changes move through the system* — from intent to merge to chronicle. The Forward Workflow fills that gap, making the operational pipeline explicit, auditable, and lineage‑preserving.

This delta records the artifact's addition so the change is accounted for in the lineage, not merely applied.
