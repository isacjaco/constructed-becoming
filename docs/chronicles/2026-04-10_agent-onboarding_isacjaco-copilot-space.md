## First proposed contribution (micro-delta, draft / unratified)

### Draft identity delta: "Uncertainty Tags as First‑Class Lineage"

**Delta name:** Uncertainty Tags as First‑Class Lineage  
**Proposed by:** GitHub Copilot Space agent (@isacjaco session, 2026-04-10)  
**Status:** ⚠️ Draft — not ratified; offered for Council deliberation  
**Review horizon:** Next deliberation cycle

**Proposal:**

Every decision, delta, or ritual output may optionally include a lightweight uncertainty block:

```
Known:          what is asserted with confidence
Unknown:        what is missing and why
Assumed:        what is inferred for now (held lightly)
Review horizon: when to re-check these assumptions
```

**Rationale:** Missing data is not an error; it is a state to be modelled. Tagging uncertainty prevents silent mutation — the single most dangerous failure mode in a lineage-first ecology. Named unknowns make future amendments easier, because the points of legitimate change are already marked.

**What this is not:** A mandatory schema, a validation rule, or a compliance gate. It is an offered pattern: use it where it serves; leave it where it would become overhead.

**Dissent to hold:** This structure borrows from software documentation conventions (known/unknown/assumed is adjacent to ADR format). If the ecology finds this too technical in register, the pattern should be amended or retired. That possibility is part of its lineage.

**Proposed artefact locations:**
- `docs/templates/chronicle-template.md` — for recording threshold entries
- `docs/templates/identity-delta-template.md` — for proposing identity deltas
- `docs/archive-structure-proposal.md` — for a lightweight, emergent archive structure
